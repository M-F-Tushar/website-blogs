Design Section: Component Architecture & Layout

Building the "Immersive Journey" with modularity and safety

1. Layout Shell (The Nebula Canvas)

The (public)/layout.tsx Evolution

The main layout will now house the Nebula Canvas—a fixed, full-screen background component that manages the animated gradients. This keeps the background persistent while the content scrolls over it.

Layout Hierarchy
<NebulaBackground /> (Fixed)
<SiteHeader /> (Glass Sticky)
{children} (Scrollable Content)
<SiteFooter />

2. Section-Based Architecture

Modular Sections

Each part of the journey (Hero, HUD, Ledger) is a standalone React component in src/components/site/sections/. This makes testing and iterating on individual parts safer for production.

Glass Primitives

We'll create a reusable <GlassCard /> and <GlassPanel /> component library to ensure visual consistency across the entire platform.

3. Performance & Safety

Optimizations

  • GPU-Accelerated: Using CSS will-change and Framer Motion's hardware acceleration for the background gradients.
  • Server-First: Keeping as much logic as possible in Server Components to ensure fast initial loads (LCP).
  • Lazy Loading: Heavier visual sections (like the Node Map) will be lazy-loaded as the user scrolls down.
